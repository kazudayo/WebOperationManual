# 1.Core Components of MIDI 2.0

# 1.1 Discovery, Bidirectional Autoconfiguration, and Expanded Data Format

  • MIDI 2.0はMIDI 1.0を拡張した規格

  • 主な特長は以下。

    • 自動設定
    • 双方向通信
    • 機器の発見の仕組み
    • 将来拡張性のある高解像度データフォーマット

# 1.2 Four Core Documents

  • 以下の４つの規格書が主要ドキュメント。

      1. MIDI Capability Inquiry (MIDI-CI)

# 1.3 MIDI Capability Inquiry (MIDI-CI)

-MIDI-CIは機器発見のメカニズムと構造を定義している。

-MIDI-CIの目的は以下。

  • SenderがReceiverの能力を知る

  • 自動設定で簡単に

  • さまざまな機器情報を取得する方法を定義する

-MIDI-CIで以下の３機能が提供される。

  • Profile Configuration : Profileによる自動設定.

  • Property Exchange : JSONを利用した機器情報の取得・設定

  • Process Inquiry : Discover the current state/value of supported MIDI Messages

  • Profile ConfigurationとProperty Exchangeは独立した規格書が存在するが、Process InquiryはMIDI-CI規格書の中に含まれている。

-MIDI-CIによるDiscorveryメカニズムは他のMIDI-CI機能を実行する前に必ず動作させる。

# 1.6 Universal MIDI Packet (UMP) Format and MIDI 2.0 Protocol

  • MIDI 1.0 ProtocolメッセージとMIDI 2.0 Protocolメッセージ、拡張方法を定義

  • UMP FormatにはEndPointのDiscoveryメカニズムが含まれている

  • UMP DiscoveryによってMIDI 1.0 ProtocolかMIDI 2.0 Protocolの切り替えが可能になる

  • Groupの概念が追加される。GroupはMIDI 1.0の16ChannelとSystem Messagesをセットにしたもので最大16Group。

  • Jitter Reduction Timestampが追加。タイムスタンプをMIDIメッセージの先頭に追加して、タイミング精度を向上できる。

  • MIDI 2.0 Protocolでは高解像度のChannel Voice Messagesが利用できる

  • Channel Voice Messagesでは複数の情報を１つのメッセージに組み合わせることができる

  • 新たなChannel Voice Messagesも追加され、ノート毎のコントロールや音楽表現できるように

  • 新たなデータメッセージにはSystem Exclusive 8とMixed Data Setが追加された

  • System Exclusive 8はMIDI 1.0のSysExに非常に似ていて、8bitデータフォーマット

  • Mixed Data Setは大きなサイズのファイル転送やMIDIではないデータのやり取りに利用される

  • 新たなデータメッセージはMIDI 1.0 SMFのメタイベントに相当する情報もやりとりできる

  • 全てのメッセージはSMF2で利用可能

  • UMP Formatには将来拡張のため、大きな予約領域が存在している
