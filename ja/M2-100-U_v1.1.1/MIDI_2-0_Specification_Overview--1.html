# 1. 序文

# 1.1 仕様概要

MIDI 2.0はMIDI 1.0の拡張である。MIDI 2.0はMIDI 1.0に取って代わるものではなく、MIDI 1.0の基本的な考え方、アーキテクチャを土台として設計されている。

MIDIは、電子楽器間における通信の世界標準として1983年に誕生して以来、成長を続け、ユーザーに貢献してきた。その後、MIDI業界は一般社団法人音楽電子事業協会（AMEI）とMIDI Association（MA）による標準化の取り組みを通じて、数十年という長い年月をかけて、MIDIの機能と市場範囲を大幅に拡大してきた。

MIDI 1.0には、将来の拡張を妨げるいくつかの制約があるが、MIDI 2.0ではこれらが解決されている。MIDI 2.0のコンセプトは機器同士の自動設定や緊密な相互運用性を実現するために、双方向通信を活用することである。新たなMIDI メッセージ データフォーマットにより、より表現力豊かな音楽制御が可能になり、将来的に新しい MIDI メッセージを追加できるようになる。

本仕様では、MIDI 2.0の基本仕様とMIDI 2.0対応機器と謳うための最小要件を記載する。

# 1.2 MIDI 2.0概要

本仕様では、MA/AMEIが発行する規格のうち、MIDI 2.0の基本メカニズムに属する規格群を定義する。MIDI 2.0は独立した規格ではない。MIDI 2.0を実装するには、メーカーと開発者はMIDI 1.0を理解しておく必要がある。

また、機器が MIDI 2.0互換性を主張するための最小要件も定義している。

# 1.2.1 MIDI 2.0の基本仕様

本仕様と併せて、下記の仕様においてMIDI 2.0 の基本仕様を定義している。

  1. M2-101-UM MIDI Capability Inquiry (MIDI-CI), Version 1.2 [MA02]

  2. M2-102-U Common Rules for MIDI-CI Profiles, Version 1.1 [MA03]

  3. M2-103-UM Common Rules for Property Exchange, Version 1.1 [MA04]

  4. M2-104-UM Universal MIDI Packet (UMP) Format and MIDI 2.0 Protocol, Version 1.1 [MA05]

これらの仕様については、セクション 2 にて説明する。

# 1.2.2 MIDI 2.0の最小要件

MIDI 2.0の最小要件を満たすためには大別すると2つの方法がある。
MIDI Capability Inquiry (MIDI-CI) [MA02] とそれ付随する機能、あるいは/加えて Universal MIDI Packet (UMP) [MA05] とそれ付随する機能を機器に実装しなくてはならない。

最小要件の完全な定義については、セクション 5 を参照。
