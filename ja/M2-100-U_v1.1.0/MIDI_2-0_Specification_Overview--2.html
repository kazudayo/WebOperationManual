# 2. MIDI 2.0のコアコンポーネント

# 2.1 検出、双方向自動構成、および拡張データ形式

MIDI 2.0 は、MIDI 1.0 の中核となる原則、アーキテクチャ、セマンティクスに基づいて構築されている。主な機能は、双方向接続によって有効になり、デバイスが他の接続されたデバイスの詳細を検出できるようにする自動構成と、将来多くの新しいメッセージを定義できる拡張性を備えた高解像度の拡張データ形式である。

  • MIDI-CIにより、デバイスが相互に検出し、それぞれがサポートする MIDI 2.0 自動構成メカニズムを学習できるため、相互運用性が向上する。

  • Universal MIDI Packet (UMP)は拡張データ形式を定義する。新しいメッセージにより、デバイスは互いのトポロジと、高解像度メッセージで通信するためのデータ サポート機能を検出できるようになる。

これらの機能の基本的な定義は、セクション 2.2 にリストされている仕様に記載されている。

# 2.2 4 つの主要文書

MIDI 2.0 規格は、MIDI 1.0の拡張機能を記した複数のドキュメントで構成されている。これらのドキュメントは、MIDI 2.0のコアアーキテクチャおよびMIDI 1.0との関係性を定義している。また、MIDI 1.0を踏まえたドキュメントとなっている。
メーカーと開発者は MIDI 2.0 を実装するには、MIDI 1.0 を十分に理解する必要がある。

MIDI 2.0の各種特徴は、以下のドキュメントで定義される。

  1. MIDI Capability Inquiry (MIDI-CI) [MA02]

  2. Common Rules for MIDI-CI Profiles [MA03]

  3. Common Rules for MIDI-CI Property Exchange [MA04]

  4. Universal MIDI Packet (UMP) Format and MIDI 2.0 Protocol [MA05]

# 2.3 MIDI Capability Inquiry (MIDI-CI)

MIDI-CI は、双方向通信を行うデバイスが自動構成と相互運用性のために MIDI 2.0 の拡張機能を使用することに同意できるようにする検出メカニズムとアーキテクチャを定義する。

MIDI-CIのねらい:

  • Senderは、ReceiverがどのMIDI 2.0の機能に対応しているかを知ることができる。

  • 接続されたDevice間の連携を自動で設定することができる。

  • 様々なDeviceのProperty Dataを取得、設定する方法を定義する。

これらを実現するための3つの機能:

  • Profile Configuration: Device間の連携を自動設定。

  • Property Exchange: JSONフォーマットによるDeviceのProperty Dataの取得・設定。

  • Process Inquiry:サポートされている MIDI メッセージの現在の状態および値を検出。

MIDI-CI は、profile設定とproperty交換のコア メッセージを定義する。profile設定とproperty交換の実装に関する詳細とルールについては、「共通ルール」ドキュメント、セクション 2.4 および 2.5 を参照。

MIDI-CI 仕様には、プロセス問い合わせの完全な仕様が含まれているが、プロセス問い合わせに関する個別の共通ルール文書はない。

MIDI-CI の他の機能が動作する前に、MIDI-CI 検出メカニズムを確立する必要がある。

詳しくは、「MIDI Capability Inquiry (MIDI-CI)」を参照。

# 2.4 Common Rules for MIDI-CI Profiles

Profileは、自動設定を可能にする便利なコンポーネントである。

MIDI-CI Profileは、特定の目的を達成するため、または特定のアプリケーションに適合するために定義された MIDI メッセージと実装ルールのセットである。 MIDI メッセージへの応答の定義に加えて、Profileではオプションで他のデバイス機能要件も定義できる。一般に、Profileはレシーバーの実装を定義する。Profile定義は、Sender の MIDI 実装を暗黙または要求する場合もある。

Profileは、MIDI 1.0におけるGeneral MIDIと類似している。GMを利用することで、プログラム・チェンジで選択可能な音色セットが使用できること、16MIDIチャンネル全てが受信できること、チャンネル10がドラムセットであること、選択されたMIDIメッセージのセットに対する定義済みのレスポンスがあることを、Deviceに認識させることができる。

このような仕様をDevice間で共有することで、送信するMIDIメッセージに基づいたDeviceの動作を予測して、Device間の連携を可能にした。

MIDI-CI Profileは、より統合された連携をDevice間で実現することを目指している。GMは、Profileに近いコンセプトモデルだったが、MIDI-CIや双方向通信のコンセプトは存在しなかった。GMでは、「GM On」メッセージが存在したが、Receiverからの応答は存在しなかった。MIDI-CIでは、双方向通信が可能となる。Common Rules for MIDI-CI Profilesでは、Profileの具体的な記述方法や、DeviceにおけるProfileの使用方法を定義している。

詳しくは、「Common Rules for MIDI-CI Profiles」を参照。

# 2.5 Common Rules for MIDI-CI Property Exchange

Property Exchangeとは、MIDI-CIのUniversal System Exclusiveメッセージを用いて送受信されるJSONによってDeviceのProperty Dataの取得・設定を行うメカニズムである。

Property Exchangeを使用すると、Deviceに対する深い知識や、独自のソフトウェア開発を必要とせず、Deviceのコントローラーの自動マッピング、プログラムの選択、状態変更、さらにはDAWへのビジュアルエディターの提供などを行えるようになる。これにより、デスクトップOS、モバイルデバイス、Webブラウザーなど、幅広いシステム上で動作するDeviceと、DAWやハードウェアコントローラーとのスマートな連携が可能になる。

Property Exchangeは、Property Dataの送信方法を定義するスキーマによって、Deviceが連携するための共通の方法を提供する。

Property Exchangeを使用することで、カスタムソフトウェアとSysExを用いた独自のソフトウェアで、さらなる独自の体験を提供することが可能になる。さらに、機器の耐用年数やアクセシビリティ、システムのアップグレードや新たなプラットフォームによる制約を受けることがなくなる。

詳しくは、「Common Rules for MIDI-CI Property Exchange」を参照。

# 2.6 Universal MIDI Packet (UMP) Format and MIDI 2.0 Protocol

Universal MIDI Packet (UMP) Format and MIDI 2.0 Protocol規格は、MIDI 1.0 ProtocolメッセージとMIDI 2.0 Protocolメッセージの新しいデータフォーマットと MIDI 拡張の基盤を定義する。

UMP Formatには、Universal MIDI Packet (UMP) Formatおよび MIDI 2.0 Protocol仕様の他の機能を使用する前に制定する必要がある UMP エンドポイントの検出メカニズムが含まれている。 UMP ディスカバリにより、MIDI 1.0 Protocolと MIDI 2.0 Protocolを切り替えるための双方向トランザクションなど、相互運用性が向上する。

UMP のデータ形式では 16 のグループという概念が新たに加わっている。各グループには独立したシステム・メッセージと、MIDI 1.0のチャンネルに相当する16チャンネルが含まれている。Universal MIDI Packet formatにはJitter Reduction Timestampのメカニズムが含まれており、タイミング精度を向上するために、MIDIメッセージの先頭にタイムスタンプを追加できる。

MIDI 2.0 Protocolは、すべてのチャンネル ボイス メッセージに対して拡張されたデータ解像度を提供する。 MIDI 2.0 では、組み合わせメッセージを 1 つのアトミック メッセージに集約することで、一部のメッセージを使いやすくしている。いくつかのチャネル ボイス メッセージに新しいpropertyを追加する。新しいチャンネルボイスメッセージが追加され、ノートごとのコントロールと音楽表現が向上した。

新しいデータ・メッセージには、System Exclusive 8とMixed Data Setが含まれる。System Exclusive 8は、データフォーマットが8ビットであることを除き、MIDI 1.0システム・エクスクルーシブと非常によく似ている。Mixed Data Setメッセージは、MIDI以外のデータを含む大きなデータ・セットの転送に使用される。

新しいメッセージは、標準 MIDI ファイル形式のバージョン 1 では以前はnon-MIDI「メタ イベント」であったpropertyを表現するために定義されている。標準 MIDI ファイル形式のバージョン 2 のすべてのpropertyは、UMP 形式の MIDI メッセージと交換できる。

Universal MIDI Packet formatは、将来的な拡張のために、多くのリザーブ領域がある。

詳しくは、「Universal MIDI Packet (UMP) Format and MIDI 2.0 Protocol」を参照。
